概述

《乱战机动队》（英文名：Past Due）是一款由零壹互动娱乐科技(北京)有限公司开发的多人在线战术竞技射击游戏。我在该项目的主要职责是关卡设计。

项目信息

  • 平台：PC/Steam
  • 类型：TPS，Battle Royale
  • 引擎：Unreal Engine 4.x
  • 团队人数：35+
  • 使用过的工具：Photoshop,SketchUp,Blender
  • 官网:

职责

  • 独立负责设计游戏大地图关卡的文档撰写。
  • 设计关卡的2D Layout，制作并搭建关卡白盒，协同美术创造场景的 3D 资源。
  • 部分Landscape的编辑迭代。
  • 从概念到可玩关卡，全程跟进关卡的制作与实现，根据反馈，不断迭代关卡，确保关卡符合游戏的预期体验。
  • 根据主策的想法完成核心模式的文档撰写，完成该模式的详细设定，跟进程序与美术制作该功能与其相关模型。

前期制作

本游戏在战斗方面更偏向于中近距离、立体机动的战斗风格，TTK较长。根据游戏的基础设定、核心玩法以及预期的游戏体验，从多个的角度来确定关卡的制作标准和layout。以下几点描述了部分主要内容：

  • 地图大小：单局游戏时长的预期（20~ 30分钟），单局人数（60人），玩家的基础移动速度（6m/s），子场景大概数量和大小，核心玩法（BR为基础），主要通过这几点确定了地图的尺寸为2.5km x 2.5km，实际的游玩范围在近似于2km为直径的圆内。

  • 地形（LandScape）：游戏偏向于中近距离作战，我们希望游戏的交战距离在250m以内，主要发生于0~ 200m之间。所以资源点及周边地区的视野并不会太广阔，通过地形将不同子场景在视野上分隔开，子场景多出现在高度较低的地区，同时这样对游戏性能上的优化也有帮助。野外通过树林、石头与起伏的地形使玩家在野外的交战距离缩短，减少远距离的交战可能性。

  • 子场景（资源点）的分布：考虑到玩法会促使玩家逐渐向地区中心移动，高等级资源点主要分布在地图边缘。资源点主要分为两个维度：密度以及等级，这两点共同决定了区域的富有程度。作为主要资源点的子场景数量和规模大小设定为能够符合3人一组共20组的玩家在游戏中的成长节奏，以及预期的交战频次。次要的资源点一般设立在主要资源点之间的必经之处。

  • 角色属性：设定角色的部分属性，如不同状态下的角色高度、枪口高度、跳跃高度、飞行高度以及Walkable的角度等，为后续的场景制作奠定基础。

  • 场景制作标准：设定房屋建筑的门窗大小、房屋高度及最小的室内面积，不同掩体的高度、宽度及其他属性，场景物件的约束等，全地图的资源按照这个标准来制作，确保场景是可玩且拥有统一标准的。

制作

这部分主要描述子场景的设计、制作与其迭代。

  • 子场景：同样，子场景也根据游戏的预期体验等不同角度进行设计。为了丰富地图的乐趣，我们为每个主要子场景都进行了不同程度上的单独设计。

    1. 特殊地貌地形
      地图中共有3处特殊地形的场景。分别是采矿场、峡谷和沼泽地,均是在生成地形的基础上进行再修改。

      • 沼泽地视野开阔，有大量中距离的作战环境，但也更容易发生多方混战。
      • 采矿场场景结构立体，有较大的高低差，由数个矿坑及狭长的道路组合而成，每个矿坑相当于一个小的作战区，中心区域是整个子场景的高地区域，视野良好可以观察到周边地区的情况。通过特殊的水晶、植被与地表在全地图中辨识度非常高。
      • 峡谷同样结构立体，在多个高地之间有直连的桥梁或自然形成的道路，供玩家在高地之间快速行进移动，峡谷之下道路狭长且多岔路，更加隐蔽，但更容易受到高地的攻击。

    2. 次要子场景
      次要的子场景遍布在主要资源点以外的地区，填补空白给予更多的中近距离的战斗场景，其中部分设立在主要资源点之间的必经之处。共设计了4种大小和结构不同的室内建筑，下方的这些截图展示了从白盒到最终游戏中的场景的整个过程。

    3. 独立设计的大型建筑
      配合游戏背景，设计了四处拥有独特地貌和建筑风格的大型建筑场景。在美术方面，玩家可以从远处就看到标志性的建筑外形，分别设计了不同的剪影风格作为了LandMark，在关卡的可玩性方面，每一个场景都经过了单独设计，类似于其他FPS中TDM模式中的小地图关卡，分别提供不同的战斗场景，以及战术战略上的选择，并且加入了跳板这一趣味性的设计，供玩家在立体空间中战略选择上更加丰富。

      • 三角遗迹：layout以中心向不同的3个方向延伸，在结构上呈现多道路双层结构，是整张地图立体空间最多的地方。
      • 断崖遗迹：战斗区域更加封闭，只有3个入口，入口易守难攻，但同时该区域附近场地空旷更难转移到其他地区。断崖处提供了远距离的跳板，供玩家无视地形更方便的转移位置。内部结构被分割成多个小空间，更考验玩家的反应速度和技巧。
      • 山顶遗迹：覆盖了一条连接工厂和矿坑之间的公路与其两侧的山坡，被覆盖的公路变成了两层结构，较高的地势提供了更好的视野。
      • 遗迹4：4个当中占地面积最小，结构对称且紧凑，共有3层区域。近距离遭遇战较多，且对机动性高的角色非常有利。该子场景连接高耸山崖，对两侧形成了阻碍行进的地形，且是该山崖唯二的出入口。

    4. 建筑群
      地图设计4处可复用建筑的建筑群，其中工厂偏向于中近距离的巷战，贫民窟结构多层立体，结构较复杂，另外两处乡镇建筑分散，在地形上更加多变，下方截图展示了这些子场景。

事后回顾

  • 以上描述的关卡设计是最新版本的地图，这将近2年的开发当中，除了关卡本身的迭代，关卡根据玩法也改过数次。考虑到时间与成本，关卡都是基于前一个版本进行改动的，关卡设计也因此而有所取舍。如果能够不考虑这些因素，关卡方面仍然有改进的空间。

  • 子场景虽然都有经过测试迭代，能够符合游戏玩法和预期体验，但某些子场景显得有些平庸。我认为应该在早期的关卡设计阶段，更加不受限制的多设计不同的场景原型（例如2倍于所需场景数量），多尝试有趣的点，在测试迭代中剔除平庸和不合适的场景，最终选出所需数量的场景。

  • 游戏的版本后续情况，在游戏上线后，可以根据测试数据分析玩家地图的游玩情况在进行重新设计与调整。这样也可以让同一张地图的生命力更长久。

  • 部分设计文档、layout以及早期的prototype的资料没有存档，所以有些只用了文字来描述，没有图片来辅助说明。
